🌌 OrchX: Universal AI Automation AgentOrchX 是一個基於 .NET 8.0 開發的高性能 C# AI Agent 系統。它不僅能與 Google Gemini API 進行對話，更核心的設計在於其「工具調度」與「多專家協作」能力，打造一個能自我優化、具備環境感知能力的自動化工作站。✨ 核心特色1. 多專家協作生態 (Multi-Agent Ecosystem)透過 consult_expert 工具，主 Agent 可動態建立多位領域專家（如安全性專家、架構師、程式碼審查員）：獨立記憶：每位專家擁有獨立的對話歷史與上下文。深層對話：支援同步或非同步任務指派，並能進行多輪深度的往復討論。2. 強大的模組化工具箱 (Modular Toolset)採用插件式架構，易於擴充：智慧檔案系統：支援樹狀目錄檢視、大檔案自動摘要（Fast Model）及精準的單行修改 (update_file_line)。環境與網路：整合完整的 HTTP GET/POST 請求，並具備終端指令執行能力（受控的白名單模式）。技能學習：AI 可自行建立與更新 .agent/skills 下的 SOP 規範，實現能力的持續增長。3. 自動化與自癒能力Mock API 模式：在無 API Key 環境下可自動回放 MockData/ 內容，方便開發測試。歷史壓縮：當對話超過 Token 閾值（預設 80 萬）時，自動觸發摘要壓縮以節省空間。異常恢復：發生嚴重錯誤時，系統會自動備份對話紀錄至 JSON 檔案。📁 專案結構PlaintextOrchX/

🌌 OrchX: Universal AI Automation AgentOrchX is a high-performance AI Agent system built on .NET 8.0. More than just a chatbot for Google Gemini, its core design focuses on "Tool Orchestration" and "Multi-Agent Collaboration," creating a self-optimizing, environment-aware automation workstation.✨ Key Features1. Multi-Agent EcosystemThrough the consult_expert tool, the main Agent can dynamically spawn specialized experts (e.g., Security Expert, Architect, Code Reviewer):Independent Memory: Each expert maintains its own isolated conversation history and context.Deep Reasoning: Supports both synchronous and asynchronous task assignments for rigorous back-and-forth discussions.2. Powerful Modular ToolsetBuilt with a pluggable architecture for seamless expansion:Smart File System: Features tree-view exploration, automatic large-file summarization (via Fast Model), and precise line-by-line updates (update_file_line).Environment & Network: Integrated HTTP GET/POST requests and terminal command execution (via a controlled whitelist mode).Skill Acquisition: The AI can create and update SOPs under .agent/skills, allowing for continuous capability growth.3. Automation & Self-HealingMock API Mode: Automatically replays content from MockData/ when no API Key is available, facilitating offline development.History Compression: Automatically triggers summary compression when the conversation exceeds token thresholds (default 800k) to save context space.Failure Recovery: Automatically backs up chat history to JSON files during critical errors or user interruptions.📁 Project StructurePlaintextOrchX/
